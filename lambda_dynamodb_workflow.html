SAM + Docker + OIDC 개발 워크플로우

액세스 키 없이 로컬 개발부터 클라우드 배포까지 완벽하게 처리하는 방법

로컬 테스트 (Local)

  • ✅ Docker: DynamoDB Local 실행
  • ✅ SAM CLI: 로컬 네트워크 연동
  • ✅ 권한: 별도의 AWS 키 필요 없음

운영 배포 (GitHub Actions)

  • ✅ OIDC: 임시 보안 자격 증명 사용
  • ✅ SAM Deploy: 자동화된 배포 파이프라인
  • ✅ 보안: 유출 위험이 있는 Key 미사용
1

도커 설정 (docker-compose.yml)

로컬에서 실행될 DynamoDB 컨테이너를 정의합니다. SAM 람다와 통신할 수 있도록 동일한 네트워크를 사용해야 합니다.

version: '3.8'
services:
  dynamodb-local:
    image: amazon/dynamodb-local
    container_name: dynamodb-local
    ports:
      - "8000:8000"
    command: "-jar DynamoDBLocal.jar -sharedDb"
    networks:
      - lambda-net

networks:
  lambda-net:
    driver: bridge
2

람다 코드 작성 (Node.js SDK v3)

로컬 환경인지 실제 람다 환경인지 판단하여 DynamoDB 엔드포인트를 자동으로 전환합니다.

import { DynamoDBClient } from "@aws-sdk/client-dynamodb";

// SAM 로컬 실행 시 AWS_SAM_LOCAL 환경 변수가 자동으로 세팅됨
const isLocal = process.env.AWS_SAM_LOCAL === 'true';

const client = new DynamoDBClient({
  region: "us-east-1",
  // 로컬일 경우 docker-compose의 서비스 이름을 엔드포인트로 설정
  endpoint: isLocal ? "http://dynamodb-local:8000" : undefined,
});
3

로컬 실행 명령어

터미널에서 아래 명령어를 순서대로 실행하여 로컬 테스트를 진행합니다.

# 1. 도커 네트워크 생성 및 DB 실행

docker-compose up -d

# 2. SAM 로컬 람다 호출 (네트워크 옵션 필수)

sam local invoke "FunctionName" --docker-network lambda-net

4

배포 환경 (GitHub Actions)

이미 설정된 OIDC를 통해 `sam deploy`가 실행되므로, 별도의 키 없이 안전하게 배포됩니다.

# .github/workflows/deploy.yml 중 일부
- name: Configure AWS credentials
  uses: aws-actions/configure-aws-credentials@v4
  with:
    role-to-assume: arn:aws:iam::123456789:role/github-oidc-role
    aws-region: us-east-1

- name: SAM Deploy
  run: sam build && sam deploy
문서 포털로 돌아가기

💡 요약

액세스 키가 없어도 로컬은 Docker-Compose로 실제 DB를 흉내 내고, 배포는 OIDC로 IAM Role을 위임받아 수행하면 됩니다. 이 방식은 보안성이 가장 뛰어나며 실제 운영 환경과 코드 차이를 최소화하는 전문적인 개발 방식입니다.

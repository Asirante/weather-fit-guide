Docker + OIDC 개발 워크플로우

액세스 키 없이 로컬 개발부터 클라우드 배포까지 완벽하게 처리하는 방법

⛔

SAM CLI 사용 불가 (환경 제약)

학교 AWS 계정의 비용관리 태그 정책으로 인해 sam build / sam deploy / sam local invoke 명령어는 우리 환경에서 사용할 수 없습니다. 로컬 테스트는 아래 Step 3의 직접 실행 방법을, 배포는 Step 4의 AWS CLI 방식을 사용하세요.

로컬 테스트 (Local)

  • ✅ Docker: DynamoDB Local 실행
  • ✅ Python 직접 실행: 핸들러 함수 로컬 호출
  • ✅ 권한: 별도의 AWS 키 필요 없음

운영 배포 (GitHub Actions)

  • ✅ OIDC: 임시 보안 자격 증명 사용
  • ✅ AWS CLI: lambda update-function-code
  • ✅ 보안: 유출 위험이 있는 Key 미사용
1

도커 설정 (docker-compose.yml)

로컬에서 실행될 DynamoDB 컨테이너를 정의합니다. Lambda 코드와 통신할 수 있도록 동일한 네트워크를 사용합니다.

version: '3.8'
services:
  dynamodb-local:
    image: amazon/dynamodb-local
    container_name: dynamodb-local
    ports:
      - "8000:8000"
    command: "-jar DynamoDBLocal.jar -sharedDb"
    networks:
      - lambda-net

networks:
  lambda-net:
    driver: bridge
2

람다 코드 작성 (Python boto3)

로컬 환경인지 실제 Lambda 환경인지 판단하여 DynamoDB 엔드포인트를 자동으로 전환합니다.

import boto3
import os

# 로컬 실행 시 IS_LOCAL=true 환경변수를 직접 설정
is_local = os.environ.get('IS_LOCAL') == 'true'

dynamodb = boto3.resource(
    'dynamodb',
    region_name='us-east-1',
    # 로컬일 경우 docker-compose의 DynamoDB Local 엔드포인트 사용
    endpoint_url='http://localhost:8000' if is_local else None
)
3

로컬 실행 방법

SAM CLI 대신 Python을 직접 실행하거나 pytest로 테스트합니다.

# 1. DynamoDB Local 실행 (Docker)

docker-compose up -d

# 2. 환경변수 설정 후 핸들러 직접 실행

IS_LOCAL=true python -c "from lambda_function import lambda_handler; lambda_handler({}, None)"

# 3. pytest 단위 테스트

IS_LOCAL=true pytest tests/

4

배포 환경 (GitHub Actions — AWS CLI)

OIDC로 인증 후 aws lambda update-function-code로 코드를 직접 업데이트합니다. Lambda 함수 자체는 콘솔에서 미리 생성되어 있어야 합니다.

# .github/workflows/deploy-backend.yml 예시
- name: Configure AWS credentials
  uses: aws-actions/configure-aws-credentials@v4
  with:
    role-to-assume: ${{ secrets.AWS_ROLE_ARN }}
    aws-region: us-east-1

- name: Zip and Update Lambda Code
  run: |
    cd backend
    zip -r ../deploy.zip .
    aws lambda update-function-code \
      --function-name inhatc-team2-1-recommendAPI \
      --zip-file fileb://../deploy.zip
사전 조건: Lambda 함수(inhatc-team2-1-recommendAPI 등)는 반드시 AWS 콘솔에서 미리 생성되어 있어야 합니다. 환경변수, 트리거, IAM Role 등도 콘솔에서 직접 설정하세요. 자세한 내용은 하이브리드 배포 가이드를 참고하세요.

💡 요약

액세스 키가 없어도 로컬은 Docker-Compose로 DynamoDB를 실행하고 Python으로 직접 핸들러를 테스트합니다. 배포는 OIDC로 IAM Role을 위임받아 AWS CLI(lambda update-function-code)로 수행합니다. SAM CLI는 학교 환경에서 사용할 수 없으며, Lambda 인프라 설정은 콘솔에서 직접 진행합니다.

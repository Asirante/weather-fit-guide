Lambda (수집용) EventBridge S3 (Raw Data) DynamoDB (기상/대기 캐시)

    inhatc-team2-5-dataAPI 및 inhatc-team2-5-real-time-dataAPI를 활용하여 파이썬 코드로 날씨/대기질 데이터를 수집합니다.

    inhatc-team2-data-trigger 및 inhatc-team2-real-time-trigger를 통해 rate 기반으로 스케줄이 동작하도록 설정합니다.
  • ✅ 데이터 적재 (S3): API에서 받아온 원본 JSON 데이터를 S3 버킷(inhatc-team2-5-raw-data)에 누적 저장합니다.

Lambda (서버용) Function URL DynamoDB (AI 캐시) Bedrock (API 연동)

  • ✅ 엔드포인트 생성: API Gateway 대신 Lambda Function URL을 활성화하여 프론트엔드가 호출할 수 있는 HTTP 주소를 만듭니다.

    team2-recommend-cache를 활용하여 프론트의 요청이 오면 기상패턴을 Key로 이 테이블을 조회하고, 캐시 히트 시 즉시 반환합니다.
  • ✅ AI 호출 및 적재: 캐시(DB)에 없는 신규 날씨인 경우, Bedrock API를 호출해 추천을 받고, 이 결과를 DB에 저장한 후 프론트에 반환합니다.

UI/UX 및 CDN 보안 인프라 구축

활용할 AWS 서비스

S3 (Web Hosting) CloudFront Route53 ACM (HTTPS) WAFv2

실무 수행 리스트 (To-Do)

  • ✅ 정적 웹 배포: React 등으로 개발한 프론트엔드 빌드 파일을 S3 버킷(inhatc-team2-3-frontend)에 업로드하여 호스팅의 뼈대를 잡습니다.
  • ✅ CDN 연동: CloudFront를 생성하고 OAC(Origin Access Control)를 설정하여 S3에 직접 접근을 막고 엣지 캐싱을 적용합니다.
  • ✅ 도메인 & HTTPS: Route53에서 커스텀 도메인을 연결하고, ACM에서 발급받은 인증서를 CloudFront에 붙여 HTTPS 보안 처리를 합니다.
  • ✅ 보안 방화벽: WAF(웹 방화벽)를 생성하여 CloudFront 앞단에 배치, 악성 요청(DDoS 등)을 차단합니다.

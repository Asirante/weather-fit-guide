Phase 1: MVP 우선 순위

👨‍💻 데이터 엔지니어(김호건) MVP Action Plan

데이터 수집 파이프라인(Lambda + EventBridge + DynamoDB/S3)은 성공적으로 구축 완료된 상태입니다!
현재 단계에서는 파이프라인 모니터링과 팀원들을 위한 API 데이터 명세서(PDF) 최신화 작업만 유지하시면 됩니다.

1

통합 API 데이터 명세서 문서화 (완료/유지)

수집하고 있는 모든 도메인(Weather, Air_stats, Forecast, Air_info)의 정보를 담은 PDF 명세서를 웹(api-specs.html)에 완벽히 통합하였습니다.
추후 새로운 데이터 소스가 추가되거나 변경되면, 프론트/백엔드/분석가가 확인할 수 있도록 해당 문서를 지속 업데이트해 주시면 됩니다.

2

데이터 수집 파이프라인 모니터링 (CloudWatch)

백엔드가 DynamoDB 캐시를 조회하기 시작하므로, 스케줄러가 정상 작동하여 최신 데이터가 누락 없이 적재되는지 점검합니다.

  • API Limit 에러 체크: 공공데이터 API의 일일 호출 한도 초과(HTTP 429) 에러가 CloudWatch 로그에 찍히지 않는지 모니터링합니다.
  • DB 적재 확인: team2-weather-cache, team2-air-cache에 TTL이나 누락 없이 최신 타임스탬프가 찍히는지 점검합니다.

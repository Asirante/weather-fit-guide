Weather-Fit Project Team 2 AWS Serverless Architecture

기상 및 대기질 정보 AI 가이드라인 서비스

EventBridge, Lambda, DynamoDB, Bedrock AI를 활용한 데이터 파이프라인 및 프롬프트 캐싱 설계 구조

코어 데이터 수집 및 서비스 아키텍처

  • 데이터 수집 (매시간 정각)

    EventBridge + Lambda를 통해 4개의 API에서 기상 및 대기질 정보를 수집합니다.

  • Raw 데이터 저장

    수집된 원본 데이터는 Amazon S3(inhatc-team2-5-raw-data)에 API로 받아온 원본 그대로 적재됩니다.

  • 최신 상태 저장 (캐시 DB)

    지역코드(PK)를 기준으로 현재 날씨 상태는 team2-weather-cache에, 현재 대기 상태는 team2-air-cache에 분리 저장하여 빠른 조회를 지원합니다.

  • 프론트엔드 및 API 제공

    S3(inhatc-team2-4-frontend)에 배포된 정적 웹은 Route53을 거쳐 CloudFront(엣지 캐싱)와 WAF(보안 방화벽) 환경에서 안전하게 제공됩니다. 이후 브라우저 단에서 API Gateway 없이 Lambda URL을 직접 비동기 호출하여 데이터를 렌더링합니다.

AI 가이드라인 및 캐싱 파이프라인

기능: 대기질 및 기상 상태에 따른 의상, 마스크, 우산 착용 가이드라인과 한줄평 제공

  • 가이드라인 캐싱 (team2-ai-prompt-cache)

    기상패턴(PK)을 기준으로 캐싱용 team2-ai-prompt-cache (DynamoDB)를 운영합니다. (TTL 없음)

  • 일일 배치 프로세스 (24시간 주기)
    • 수집된 모든 데이터의 중복을 제거하여 유니크한 기상패턴 추출
    • 기존 team2-ai-prompt-cache의 캐시 데이터 초기화
    • 새로운 기상 패턴들을 Bedrock AI (프롬프트 캐싱 적용)에 일괄 전송
    • 생성된 AI 답변을 캐시 DB에 적재

상세 컴포넌트 설명

  • 코어 DB (team2-weather-cache / team2-air-cache)

    지역코드(PK)를 기준으로 현재 날씨 상태와 대기 상태를 분리된 캐시 테이블에 가볍게 저장합니다. 스토리지 최적화를 위해 조합형 데이터는 저장하지 않습니다.

  • 캐싱 DB (team2-ai-prompt-cache)

    기상패턴(PK)을 기준으로 AI가 생성한 가이드라인(옷, 마스크 등)과 한줄평을 저장합니다. (TTL 없음)

  • 동적 조립 로직 (Lambda)

    요청 발생 시 Lambda에서 team2-weather-cache 및 team2-air-cache의 데이터를 읽어와 메모리 상에서 즉시 기상패턴을 조립합니다. 이를 통해 구조 변경 시의 유연성을 확보합니다.

AI 생성 및 최적화 전략

  • 실시간 Fallback (Miss Handling)

    요청된 기상패턴이 DB B에 없을 경우, 즉시 Bedrock AI를 호출하여 결과를 생성하고 DB B에 저장 후 반환합니다.

  • 프롬프트 캐싱 (Prompt Caching)

    Bedrock AI 호출 시 공통된 가이드라인 규칙 프롬프트를 캐싱하여 응답 속도를 높이고 API 호출 비용을 절감합니다.

  • 일일 배치 처리 비용 절감

    모든 데이터를 매번 AI로 생성하지 않고, 24시간 동안 수집된 데이터 중 중복을 제거한 유니크 패턴만 일괄 생성하여 효율을 극대화합니다.
